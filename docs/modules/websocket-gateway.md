## What it is for

The station runs **one process, one WebSocket** (`/ws`). The hub is the only
thing that touches sockets; everything else in the server talks to the bus. Its
job is to:

- accept any peer (`browser`, `device`, `fake`, `task`) and learn its identity;
- turn a peer's inbound `publish` frame into a `bus.publish` so modules react;
- turn every bus message into an outbound `event` frame for the topic's
  subscribers — broadcast or **directed** to one peer/address;
- keep a live **roster** of who's connected, for the console + `/api/docks`;
- detect dead links fast (ping/pong) and protect the process from stalled peers;
- own **runtime dock binding** (claim / unclaim / displace a device into a dock
  slot), per [runtime-dock-binding.md](runtime-dock-binding.md).

## Properties

- **Singleton.** One `WebSocketGateway` per station process, constructed in
  [main.ts](../../orbit-station/server/src/main.ts) as `new WebSocketGateway(server, bus, bindings)`.
  It owns the only `/ws` `WebSocketServer`.
- **Bus-decoupled.** The hub depends on the `Bus` and an injected `DockBindings`
  interface — it imports **no module**. Modules never see sockets; the hub never
  sees module logic. This keeps the protocol layer self-contained.
- **Identity model (hello v2).** Four deliberately-separate identities per peer:
  `dock` (tenant), `component` (slot), `kind`+`build` (software/OTA target),
  `id` (hardware instance). The **address** is `(dock, component)`; state binds
  to the address, never the hardware — swap the phone in a slot and sessions
  continue. See [protocol.ts:13-28](../../orbit-station/server/src/core/protocol.ts#L13-L28).
- **Topic-scoped.** Subscriptions are exact-match on the fixed `Topic` union
  ([protocol.ts:36-48](../../orbit-station/server/src/core/protocol.ts#L36-L48)).
  One topic per module; modules own their topics.
- **Pre-hello peers are hidden.** A socket that hasn't sent a valid `hello`
  (`announced === false`) never appears in the roster and isn't a delivery
  target.

## Guarantees

These are the invariants the hub upholds for the rest of the system:

1. **Subscription gating.** A peer only ever receives `event` frames for topics
   it has explicitly `subscribe`d to. ([websocket-gateway.ts:126](../../orbit-station/server/src/core/websocket-gateway.ts#L126))
2. **Directed isolation.** A directed message (`to` / `toAddr`) reaches *only*
   the addressed peer(s) — still gated on topic subscription. `toAddr` is
   resolved against the **live** roster at fan-out, so reconnects/hardware swaps
   need nothing special on the send path. ([websocket-gateway.ts:130-136](../../orbit-station/server/src/core/websocket-gateway.ts#L130-L136))
3. **No self-echo.** A peer never receives its own publishes back
   (`peer.id === msg.source` is skipped) — this is why the dock's ~15 KB
   recognize-request photos stopped bouncing back to the phone.
   ([websocket-gateway.ts:137-141](../../orbit-station/server/src/core/websocket-gateway.ts#L137-L141))
4. **Bounded buffering.** Broadcast traffic to a peer buffered past 1 MB is
   dropped; **directed** frames (signaling, results, tool-calls) are never
   dropped — they must arrive. ([websocket-gateway.ts:142-147](../../orbit-station/server/src/core/websocket-gateway.ts#L142-L147))
5. **Fast liveness.** A silently-dead peer is terminated within ~6 s, not at TCP
   timeout. ([websocket-gateway.ts:34-39](../../orbit-station/server/src/core/websocket-gateway.ts#L34-L39))
6. **Binding is source of truth.** On `hello`, a station-side binding ALWAYS wins
   over the dock the device asserts — a console claim is never silently
   overwritten by a device's stale compiled-in `DOCK_NAME`. A device-supplied
   dock only *seeds* a binding when none exists.
   ([websocket-gateway.ts:314-319](../../orbit-station/server/src/core/websocket-gateway.ts#L314-L319))
7. **One device per slot.** A second peer claiming an occupied `(dock, component)`
   displaces the old one (newest wins); the displaced peer is told and dropped.
   Applies on both `hello` and `claim`. ([websocket-gateway.ts:325-336](../../orbit-station/server/src/core/websocket-gateway.ts#L325-L336),
   [websocket-gateway.ts:248-256](../../orbit-station/server/src/core/websocket-gateway.ts#L248-L256))
8. **Handler isolation.** The bus itself wraps every handler in try/catch
   ([bus.ts:62-68](../../orbit-station/server/src/core/bus.ts#L62-L68)), so one
   module throwing can't break fan-out to the rest.

## Wire protocol (summary)

Full contract in [protocol.ts](../../orbit-station/server/src/core/protocol.ts);
keep that file dependency-free so a firmware/app author can integrate from it
alone.

**Peer → station** (`InboundFrame`):

| `t` | Purpose |
|---|---|
| `hello` | announce identity (`role`, `id`, `dock?`, `component?`, `kind?`, `caps?`, `label?`, `build?`) |
| `subscribe` / `unsubscribe` | add/remove `topics` |
| `publish` | push a `{topic, kind, payload}` message into the bus |

**Station → peer** (`OutboundFrame`):

| `t` | Purpose |
|---|---|
| `welcome` | accepted id + `serverTime` + resolved `dock`/`component` (`dock: null` ⇒ UNCLAIMED) |
| `event` | fan-out of a topic message (`topic`, `kind`, `payload`, `ts`) |
| `error` | invalid frame, or "displaced" notice |

## Lifecycle (a device session)

```
   dial /ws ──► #onConnect: register pre-hello Peer (announced=false), capture IP
        │
   hello ─────► #onMessage: set identity; resolve dock (binding wins);
        │       derive component from kind; displace any slot collision;
        │       send welcome; announce peer-joined on `station`
        │
   subscribe ─► add topics to peer.topics
        │
   publish ───► bus.publish({…, source: id})  (heartbeat also updates links/build)
        │
   bus '*' ───► matching event frames sent back (subscription + directed + no-echo + backpressure gates)
        │
   ping/pong ─► missedPongs sweep; reset on any frame
        │
   close ─────► remove peer; if announced, announce peer-left on `station`
```

## Usage (from `main.ts`)

The hub is constructed once and handed to modules as a `() => hub` accessor (so
roster-dependent wiring can be built after it):

```ts
const bindings = new BindingStore();              // deviceId→dock source of truth
const hub = new WebSocketGateway(server, bus, bindings);

const directory = new Directory(() => hub.roster());     // dock directory off the roster
modules.push(docksModule(directory, () => hub, bindings));
modules.push(bodylinkModule({ directory, motion, getHub: () => hub }));
modules.push(otaModule(() => hub));               // version-compare vs live roster
```

The hub's `roster()` is the live-peer source the directory, OTA, bodylink, and
the perception hub's `resolveDock`/`dockReady` all read.

## Boundaries (what it does *not* do)

- **No business logic.** It moves frames; modules decide meaning.
- **No persistence.** The roster is in-memory; the deviceId→dock binding it
  reads/writes is owned by an injected `DockBindings` store, not the hub.
- **No media.** WebRTC A/V does not flow through the hub — only the `media`
  topic's SDP/ICE **signaling** does. The media plane is the SFU; the
  [Perception Processing Hub](perception-processing-hub.md) taps that.
