# Thin-client consolidation — e2e test plan

Verification plan for the [thin-client consolidation](../decision-traces/thin-client-consolidation.md)
cuts. Run this **once, end-to-end, against a real dock** after building — the automated
tests already pass (station 590, Android unit + compile green); this plan proves the
*user-perceivable* behaviour didn't regress, per the "verify user-perceivable outcomes"
lesson (state-level green ≠ works for a human).

**What changed, so what could break:**
- Cut 1 removed **faceFollow** (idle face-tracking) + `face-track` op + `find_person` tool.
- Cut 2 removed **on-device FER emotion** + mirroring UI + `perceive.emotion` forwarding.
- Both re-homed idle-moods' presence read onto a new **`face-present`** op.

The risk surface is: idle personality (moods), presence gating, the perceive/geometry
stream, visual_search (the acquisition path that stays), and the LLM prompt grounding.

## How to run — minimize human involvement

Most of this is drivable from the laptop; **I can drive it** with the existing skills and
adb. Flag where a human is genuinely required (marked 👤).

- Build + sideload: `cd node-dock/app && ./gradlew :app:installDebug`
- Station up: `cd orbit-station && npm run dev`
- Drive turns over adb: `adb shell am broadcast -a dev.orbit.dock.debug.SAY -e text '…'`
- Inject a face sighting (emulator/headless): `adb shell am broadcast -a dev.orbit.dock.debug.FACE -e x 0 -e y 0` (note: the `-e emotion …` arg was **removed** — see T7)
- Conversation stress: the `stress-dock-pipeline` / `two-dock-converse` skills.
- Ground truth: station obs traces, `/api/docks`, `/api/brain/:dock/sessions`, task logs.
