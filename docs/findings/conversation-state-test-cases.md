## Instrumentation to bake in (so everything is testable headless)

These are built ALONGSIDE the state machine ‚Äî testability is part of the feature:

1. **`GET /api/brain/:dock/conversation`** ‚Äî the live conversation state:
   `{ mode, windowUntil, speakUntil, msToExpiry }`. The primary REST probe for
   every STATE assertion (poll it through a flow).
2. **`conversation`-state-change events on the obs/agent stream** ‚Äî every
   transition emits `{ dock, from, to, reason, at }`. Lets a test assert the exact
   transition sequence (not just the end state), and drives the phone renderer.
3. **`POST /api/brain/:dock/debug/event`** ‚Äî inject a raw conversation event
   (`tap` / `utterance{text,endedAt}` / `vad` / `tts-start` / `tts-end` /
   `connected`) at a controllable time. Drives EVERY flow headless, no mic. (Extends
   the existing `debug/say`.)
4. **Structured single-line logs** `[conv] dock mode A‚ÜíB (reason)` ‚Äî for adb/station
   log assertions when a REST poll would miss a fast transition.
5. **Phone:** the existing `DOCK_EVT` logs (face/beep) + a `present` frame the phone
   receives ‚Äî so a test can confirm the phone rendered the station's intent.

VALIDATE notes below reference these by number.

## Group A ‚Äî the basic addressed turn

### A1. Tap, speak, get a reply ‚Äî and the listening lifetime
- **UX:** tap ‚Üí beep + listening face ‚Üí "what's two plus two?" ‚Üí thinks ‚Üí speaks
  "four" ‚Üí auto-listens (C1). While listening: talking holds it; silence closes it
  after a bounded time.
- **STATE:** idle ‚Üítap‚Üí listening (window = now + `LISTEN_MS`); no VAD ‚Üí expires at
  `LISTEN_MS` ‚Üí idle; VAD ‚Üí window extends to now + `VAD_EXTEND_MS`;
  utteranceEnded(in window) ‚Üí addressed ‚Üí thinking ‚Üí speaking ‚Üí followup.
- **VALIDATE:**
  - unit: pure ConversationState with injected clock (mode + window assertions).
  - headless E2E: `debug/event tap` ‚Üí poll `GET /conversation` (1) shows `listening`
    + `msToExpiry ‚âà LISTEN_MS`; `debug/event vad` ‚Üí poll shows `msToExpiry` jumped to
    `VAD_EXTEND_MS`; `debug/event utterance` ‚Üí obs transition stream (2) shows
    listening‚Üíthinking‚Üíspeaking‚Üífollowup.

### A2. Tap, say nothing ‚Üí gives up after `LISTEN_MS`
- **UX:** tap ‚Üí listening ‚Üí silent ‚Üí after `LISTEN_MS`, face idle, off beep.
- **STATE:** listening; advance past `LISTEN_MS` ‚Üí idle.
- **VALIDATE:** unit (clock advance). E2E: `debug/event tap`, poll `/conversation`
  every 1s; assert `idle` appears at ‚âà`LISTEN_MS`; obs stream shows listening‚Üíidle
  (reason: timeout). Phone: `DOCK_EVT` shows listening-off beep.

### A3. Speak without tapping ‚Üí overheard, but STILL TRANSCRIBED
- **UX:** I chat to someone nearby ‚Üí no response.
- **STATE:** idle; utteranceEnded ‚Üí addressed=false; stays idle.
- **VALIDATE:** unit (addressed=false). Integration: feed an utterance with NO prior
  tap (real STT or `debug/event utterance`); assert (a) a **speech snapshot exists**
  via `GET /api/perception/snapshots` (transcription happened), AND (b) **no turn
  ran** (`/conversation` stays idle; session turn count unchanged).

### A4. Tap before the sentence ends ‚Üí it works (merged A4+A5)
- **UX:** I start speaking and tap before my last word ‚Üí it answers the whole
  sentence.
- **STATE:** utterance started before the tap; tap during it; utteranceEnded(after
  tap) ‚Üí addressed=true.
- **VALIDATE:** unit (ordering). E2E: `debug/event utterance{startedAt:T0}` interleaved
  with `debug/event tap{at:T0+x}` then `utterance-end` ‚Üí obs stream shows the turn ran.

## Group C ‚Äî auto re-listen (follow-up)

### C1. Reply ‚Üí follow up WITHOUT tapping ‚Äî several loops
- **UX:** ask ‚Üí answer ‚Üí beep (listening) ‚Üí follow-up NO tap ‚Üí answer ‚Üí waits
  `FOLLOWUP_MS` ‚Üí ask again ‚Üí ‚Ä¶ a multi-turn back-and-forth.
- **STATE:** speakEnd ‚Üí followup (window = now + `FOLLOWUP_MS`); utteranceEnded(in
  followup) ‚Üí addressed ‚Üí thinking ‚Üí speaking ‚Üí followup. **‚â•3-4 loops.**
- **VALIDATE:** unit (loop the transitions). E2E (no mic): `debug/event tts-end` ‚Üí
  `/conversation` shows `followup` + `msToExpiry‚âàFOLLOWUP_MS`; `debug/event utterance`
  ‚Üí turn runs; repeat 4√ó; assert each loop via the obs transition stream (2).

### C2. Reply ‚Üí silence ‚Üí stops after `FOLLOWUP_MS`
- **UX:** answer ‚Üí beep on ‚Üí quiet ‚Üí after `FOLLOWUP_MS`, beep off, idle.
- **STATE:** followup; advance past `FOLLOWUP_MS`, no utterance/vad ‚Üí idle.
- **VALIDATE:** unit + E2E: after `tts-end`, poll `/conversation`; assert idle at
  ‚âà`FOLLOWUP_MS`; phone `DOCK_EVT` off beep.

### C0. Follow-up opens after EVERY reply (decided: user- AND not-user-initiated)
- **DECISION:** the follow-up (auto re-listen) window opens whenever the dock
  finishes speaking, regardless of what triggered the turn ‚Äî a user-addressed turn,
  a proactive self-thought, or a task notification. (Simpler rule; the dock is
  always ready for a follow-up after it speaks.)
- **UX:** I ask ‚Üí it answers ‚Üí it listens (follow-up). It proactively says "I see
  you're back" ‚Üí it ALSO listens after, in case I respond.
- **STATE:** speakEnd ‚Üí followup, independent of trigger (ConversationState only
  sees speakStart/speakEnd ‚Äî it's trigger-agnostic by construction).
- **VALIDATE:** unit ‚Äî two cases: (a) tap‚Üíturn‚Üíspeak‚ÜíspeakEnd‚Üífollowup;
  (b) NO tap, turn‚Üíspeak‚ÜíspeakEnd‚Üífollowup. Both followup. (conversation-state.test.ts)
- **NOTE:** a queued self-thought defers behind the follow-up window (the user
  might be following up) ‚Äî see autonomous.test.ts.

### C3. Slow follow-up ‚Äî start late, it waits (`VAD_EXTEND_MS`)
- **UX:** answer ‚Üí I pause then start ‚Üí not cut off.
- **STATE:** followup; vadActivity before window end ‚Üí window ‚Üí now + `VAD_EXTEND_MS`;
  utteranceEnded after original `FOLLOWUP_MS` but within extension ‚Üí addressed=true.
- **VALIDATE:** unit (extend edge). E2E: `tts-end` ‚Üí wait ‚Üí `debug/event vad` ‚Üí
  `/conversation` `msToExpiry` jumps; late `utterance` still runs a turn.

## Group D ‚Äî priority

### D1. Tap is a TOGGLE
- **UX:** idle + tap ‚Üí listening; already-listening + tap ‚Üí OFF (idle). Tap flips it.
- **STATE:** idle ‚Üítap‚Üí listening ‚Üítap‚Üí idle; followup ‚Üítap‚Üí idle.
- **VALIDATE:** unit (toggle). E2E: `debug/event tap` √ó2 ‚Üí `/conversation` idle‚Üí

### D2. Leave camera during follow-up ‚Üí KEEPS listening
- **UX:** in follow-up, glance away ‚Üí conversation not abandoned.
- **STATE:** followup; face-leave does NOT clear it.
- **VALIDATE:** unit (face-leave is a no-op on followup at station). Phone
  integration: `present` mode stays listening when a FaceLost arrives during followup
  (phone-renderer test). Screenshot of the face still showing listening (optional).

### D3. Face arriving doesn't override active tap/follow-up
- **UX:** mid-conversation, a new face ‚Üí no reset.
- **VALIDATE:** unit (lower-priority arrival no-ops). E2E: in followup, inject a
  face-arrival listen request ‚Üí `/conversation` mode unchanged.

## ‚îÄ‚îÄ Reconnections (SEPARATE ‚Äî needs a simulation harness) ‚îÄ‚îÄ

> Tested TWO ways: (1) **session-level sim** `server/src/modules/brain/reconnection.test.ts`

>
> **LIVE PROCEDURE (real app, no fake peer; VERIFIED 2026-06-20):**
> ```bash
> P=anne-bot; D=78f5da3a   # dock name + adb device id
> conv(){ curl -s localhost:8099/api/brain/$P/conversation; }
> # 1. open a live window:  POST /api/brain/$P/debug/event {"event":"tap"} ‚Üí conv.mode=listening
> adb -s $D shell svc wifi disable      # 2. REAL WS drop ‚Üí station: phone online=False,
> #                                          conv reconciles to idle, window does NOT leak (R3)
> adb -s $D shell svc wifi enable       # 3. app auto-reconnects (~10s) ‚Üí re-hello
> #    4. conv = clean idle, no stale window (R2/R4); roster phone online=True again
> ```
- **R1. Lost "TTS finished":** speaking; never get `tts-end`; assert recovery ‚Äî
  `/conversation` leaves speaking by `SPEAK_MAX_MS` AND `reconcileConnected()`‚Üíidle. ‚úÖ sim
- **R2. Phone reconnect ‚Üí clean slate:** any mode; reconnect ‚Üí idle + one resync frame. ‚úÖ sim ¬∑ ‚úÖ LIVE
- **R3. Disconnect mid-listening/followup:** window doesn't leak; reconciles. ‚úÖ sim (R3 + R3b) ¬∑ ‚úÖ LIVE (wifi-drop)
- **R4. Station restart / dock re-hello:** state re-established (fresh idle + resync). ‚úÖ sim ¬∑ ‚úÖ LIVE (re-hello)
- **R5. CLEAN RESET on APP restart (user-requested):** the phone app restarts ‚Üí
  reconnects ‚Üí `hello` ‚Üí reconcile ‚Üí from ANY prior mode the conversation comes
  back CLEAN idle (no stale listening/speak window). UNIT: reconcile from every
  mode ‚Üí idle + windowUntil/speakUntil both 0 (conversation-state.test.ts). LIVE:
  restart the app; `GET /conversation` shows idle; any live face presence then
  opens its OWN window (which expires ‚Üí idle), proving it's fresh not stale.
- **R6. CLEAN RESET on SERVER restart (user-requested):** a fresh station process
  is a new ConversationState ‚Üí idle by construction; the phone reconnects into it
  via hello (reconcile + resync frame). UNIT: a new instance is idle. LIVE:
  observed every `tsx watch` reload ‚Äî phone reconnects, `/conversation` settles to
  idle (a live face window may briefly open, then expires ‚Üí idle). VERIFIED.
- **VALIDATE:** R1-R4 done via the session-level sim (`reconnection.test.ts`, 6
  cases) AND R2/R3/R4 confirmed LIVE on the real app via the wifi-drop procedure
  above (real socket close + Android reconnect). R5/R6 are
  unit-tested (`conversation-state.test.ts`) + live-verified via `/conversation`
  + the `[conv]` logs.
