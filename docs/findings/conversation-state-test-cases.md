## Instrumentation (all BUILT)

1. **`GET /api/brain/:dock/conversation`** — `{ mode, windowUntil, speakUntil, msToExpiry }`.
2. **`conversation` transition frames** on the agent/obs stream — `{ from, to, reason, at }`.
3. **`POST /api/brain/:dock/debug/event`** — inject raw events headless:
   `tap | vad | face-arrival | face-left | tts-start | tts-end | connected | utterance`.
4. **`POST /api/brain/:dock/debug/say`** — tap + final (drives a full addressed turn).
   **`POST /api/brain/:dock/debug/hear`** — a final with NO tap (the mid-turn /
   busy-queue seam; `debug/say`'s tap would interrupt).
5. **`GET /api/brain/:dock/debug/addressed`** — the addressed-decision ring (last 50):
   every heard utterance's terminal decision (`RAN-TURN`, `queue:busy`, `drain:ran`,
   `skip:*`, `stop:dismiss`, `stop:pause`, `wake…`). The primary live judge.
6. **`[conv]` one-line logs** + the phone's `DOCK_EVT` logs.
7. **`npm run -w server smoke:midturn`** — the headless E2E matrix (fake phone+body,
   real LLM): S4 + F1–F9 cover the busy queue, staleness, dismissal, pause, canned
   wake, and window-close draining. 36 assertions.

## Group A — the basic addressed turn (BUILT ✅)

### A1. Tap, speak, get a reply — and the listening lifetime
- **UX:** tap → listening face/beep → speak → thinks → answers → auto-listens.
  While you talk the window HOLDS; silence lets it close.
- **STATE:** idle →tap→ listening (`LISTEN_MS`); **STT interims hold it**
  (`speechInFlight`, rolling `INTERIM_HOLD_MS` — Addendum 8); addressed utterance →
  thinking → speaking → followup.
- **VALIDATE:** unit (`conversation-state.test.ts` A*, interim-hold cases);
  live: tap + a continuous 14s sentence stayed `listening` throughout (base 8s)
  and answered as ONE turn — verified 2026-07-13.
- **NOTE (changed):** the phone's Silero VAD no longer shapes the window — its
  `vad` frames are IGNORED station-side. `vadActivity` remains in the machine for
  `debug/event vad` and legacy tests only. The old failure (VAD-end shrinking the
  window under an in-flight utterance → "listening but ignored") is structurally
  gone.

### A2. Tap, say nothing → gives up after `LISTEN_MS` ✅ unit + E2E
### A3. Speak without tapping → overheard, still transcribed ✅
  (idle → `skip:not-addressed` in the ring; snapshot still written; the attention
  gate may act on it separately.)
### A4. Tap before the sentence ends → the whole sentence answers ✅
  (started-while-open rescue: `[openedAt − GRACE, lastWindowUntil]`.)
### A1h. An utterance that STARTED after the window closed is NOT addressed ✅
  **Load-bearing:** pins live numbers (+600ms-late start = room speech). This test
  REJECTED the trailing start-grace twice — do not re-add one (Addendum 8).

## Group B — the dock's own voice vs. others (BUILT ✅)

- **B1a. Own TTS not transcribed back** — phone AEC (primary; verified live
  2026-07-13: none of a 12-count reply's words hit the ring) + the STT echo-gate
  as opt-in fallback (`STT_ECHO_GATE=1`, default OFF so voice reflexes work).
- **B1b. Another voice DURING the dock's TTS is transcribed** — verified live:
  speech during TTS lands as `queue:busy@speaking` and is ANSWERED at settle
  (busy-queue contract). This is the barge-in foundation, now in use.

## Group C — auto re-listen / follow-up (BUILT ✅, semantics extended)

- **C0. Followup opens after EVERY reply** (trigger-agnostic). ✅ unit both ways.
- **C1. Multi-loop hands-free back-and-forth.** ✅ unit + E2E.
- **C2. Silence → idle after `FOLLOWUP_MS`.** ✅ — and entering idle fires the
  **settle chokepoint** (busy-queue drain; Addendum 4 fixes).
- **C3. Slow follow-up not cut off** — now via the interim hold (was `VAD_EXTEND_MS`).
- **C4 (NEW, Addendum 5.1). Overheard framing:** a turn born from the followup
  window (`trigger.via: 'followup-window'`) or the busy drain may STAY SILENT if
  clearly not addressed to the dock; silence opens no new window → an ambient
  chain self-terminates. Live: 3/5 meeting-style lines silenced, 3/3 genuine
  follow-ups answered (zero false silence).

## Group D — priority + deliberate closes (BUILT ✅)

- **D1. Tap is a toggle** (idle↔listening; followup→off). ✅
- **D1b/TI. Tap-to-interrupt** while thinking/speaking → abort + fresh listening
  window; a late tts-end can't clobber the tap window (`speakEnd` no-ops unless
  speaking). ✅ unit D1c–e · ✅ live. **Since app build 31 + `#interruptSpeech()`:
  the interrupt also SILENCES the phone's draining TTS queue post-turn**
  (Addendum 5.4 — before that, state changed but the speaker played on).
- **D2. Face-leave never cancels a tap/followup window.** ✅ (face windows are
  lowest priority; face-arrival is default-OFF entirely.)
- **D4 (NEW, Addendum 5.2/5.3). Spoken dismissal vs pause:** both stop the audio;
  **dismiss** ("stop / never mind / shut up / I'm not talking to you / go away")
  → ANY engaged mode → idle, busy queue cleared traced `skip:dismissed`, no
  window (re-engage via tap/palm/wake); **pause** ("wait / hold on") → abort +
  LISTENING window, queue kept. Fixture-tested lexicon (`stop-intent.test.ts`,
  incl. the live "Stop."→"So" mishear); harness F4/F8/F9; live-verified with a
  human voice. Deliberate closes clamp the addressed grace to now — no trailing
  tolerance after a dismissal.

## Voice barge-in (was PARKED — now PARTIALLY BUILT)

- **BI1. Talk over the reply:**
  - stop/pause words → the reply ABORTS (dismiss/pause above) — built + live.
  - content speech → queued (`queue:busy`) and ANSWERED at settle as one combined
    turn — built + live (the busy-queue contract; zero silent outcomes).
  - What remains unbuilt: content speech *superseding* the reply mid-flight
    (true supersede barge-in). Deliberate: ambient speech aborting the dock's own
    reply is the failure the busy queue exists to prevent.
- **BI2. No self-barge:** phone AEC verified live; `STT_ECHO_GATE=1` as fallback
  for weak-AEC devices.
