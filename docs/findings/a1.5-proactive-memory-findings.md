## DONE: auto-summarization (the always-on T2)

The documented gap (memory.md §0/§8, pyramid §9): a summary was produced ONLY on a
manual console click or a `force_get_current` tool call, so grounding's
`lastSummary` cache went stale in between. Now an **auto-summarizer**
(`auto-summarizer.ts`) fuses the recent window per active dock on a debounced
cadence and caches it — grounding stays continuously fresh.

- Cheap by construction: `shouldSummarize` fires only when ≥N new records AND the
  min interval elapsed (idle docks cost nothing; busy ones throttle). Env:
  `PERCEPTION_AUTO_SUMMARY_MS` (60s), `PERCEPTION_AUTO_SUMMARY_MIN_RECS` (3),
  `PERCEPTION_AUTO_SUMMARY=0` to disable.
- Reuses the exact summarize→cache path as force_get_current (extracted
  `summarizeWindowAndCache`), so there's one producer, not two divergent ones.
- **Tests:** 8 unit (5 decision variants + 3 orchestration incl. no-overlap),
  green 3×. Full perception+brain suites 178/178, green 2×.
- **LIVE-VALIDATED (no human):** with the app streaming, a summary appeared
  automatically — *"A person is typing on a laptop; identity not recognized"* —
  and flowed into grounding ("last summary (54s ago…)"), no manual trigger.

## SCOPED / NEEDS DECISION: summary → memory distillation

memory.md §0 says nothing auto-distills summaries into durable memories — memory
fills ONLY via the agent's `remember` tool or console. The obvious next step:
have a summary (or a session digest) ALSO write a `summary`-type, `derived`
memory with lineage.

**Why I did NOT just build it:** it's an LLM extraction step that can hallucinate
facts, and the current design (agent calls `remember` when IT judges something
durable, in context) is arguably the *better* shape — a background distiller
guessing "what's worth remembering" is lower-precision. So this is a real design
call, not a clear win.

**Q (decision):** do you want auto-distillation, and if so which?
1. **Session digest → memory** — on idle-close, the existing `#compactSummary`
   ALSO writes a `summary` memory. Bounded (one per session), low risk.
2. **Perception summary → memory** — every auto-summary also persists. Higher
   volume, more noise, needs dedup/retention.
3. **Neither** — keep agent-driven `remember` as the only writer (current design;
   tune the prompt to remember more if it under-remembers).
My lean: **(1)** if anything — bounded + useful ("what did we talk about?") — but
it's genuinely your call. Parked.

## SCOPED / NEEDS LIVE SIGNAL: gate relevance tier

transcript), and it couples to the tap-vs-auto barge-in decision (A1.4 Q1).
