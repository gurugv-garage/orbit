# A1.4 — voice barge-in: findings + open questions (for review)

> Worked autonomously while you were away. This captures what I did, what I
> self-tested, the decisions I made (and why), the **open questions** I need your

Branch: `always-on-stt`. Precondition: the A1 AEC fix (`a0b5687`) is landed — the
dock no longer transcribes its own TTS, which is what makes barge-in possible.

## The core insight

Voice barge-in is *mostly already built*:
- **Supersede** exists: `handleTurnRequest` aborts a running turn and awaits its
  unwind; `sanitizeHistory()` inserts synthetic `(interrupted)` markers
  (`session.ts`).
- An **addressed transcript already calls `handleTurnRequest`** (A1.2). So a user
  utterance arriving mid-TTS would *naturally* supersede.

**The one blocker was our own echo-gate.** The station mute (`stt-watch`,
`isSpeaking → drop audio`) deafens the station *exactly* during TTS — the window
barge-in must hear the user. With AEC now solid, that mute is both redundant
*and* blocking. So A1.4's core move is: **stop dropping audio during TTS** (let
AEC keep the dock's own voice out; hear the user for barge-in).

## What I did (autonomous)

1. **Flag-gated the echo-gate** (`STT_ECHO_GATE`, default OFF). The drop-audio-
   while-speaking mute is no longer active — AEC handles self-transcribe.
2. **Confirmed the supersede path is already complete** (`session.ts`
   `handleTurnRequest`): an addressed utterance during `speaking` aborts the
   running turn + `sanitizeHistory` inserts `(interrupted)`. So barge-in =
   "remove the gate so the station can hear the user mid-TTS"; the rest exists.
3. **SELF-TEST PASSED (no human):** gate OFF + AEC on, fired a multi-sentence
   `/think` self-reply → the station transcribed **0** speech during the dock's
   own reply (baseline 0 → 0). Proves: (a) AEC alone keeps the dock from hearing
   itself, (b) the dock will NOT self-barge (no stray self-transcript to
   supersede with). This is the negative half of barge-in, fully validated alone.

## Decisions I made (review these)

- **Removed the echo-gate as the active mechanism** (flag-gated, not deleted) —
  justified by the self-test above. Reversible via `STT_ECHO_GATE=1`.
- **Barge-in stays TAP-GATED for now** (an addressed utterance supersedes; an
  un-tapped overheard one does not). Rationale: deterministic + safe; auto-barge
  on any overheard speech risks third-party/background interruption and belongs
  with the A1.5 relevance tier. (Open Q1 — confirm.)

## OPEN QUESTIONS — need your call

1. **Tap vs. auto barge-in.** With the gate removed, the station hears the user
   during TTS. Should *any* overheard speech mid-TTS interrupt (auto), or only a
   **tapped/addressed** one? Auto is more natural but risks interrupting on
   background chatter / a third party. Leaning: **tap-gated barge-in first**
   (safe, deterministic), auto-barge as an A1.5 relevance-tier upgrade. Confirm.

2. **Echo-gate: delete or keep flagged?** I gated it behind an env flag
   (default OFF) rather than deleting, so it's a one-line revert if a device's AEC
   turns out weaker. OK to keep it flag-guarded, or do you want it gone entirely?

3. **False-barge sensitivity.** Even with AEC, a loud transient or residual could
   produce a stray transcript mid-TTS. Do we want a confidence/length floor on a
   barge-in utterance (e.g. ignore <N chars or lowConfidence), or trust the STT?

## HUMAN TEST RECIPE (run when you're back)

The live barge-in test needs a person to talk over the dock. Steps:

1. Build + install: `cd node-dock/app && ./gradlew :app:installDebug`
2. Launch the app; confirm it's streaming audio: `curl -s localhost:8099/api/media/status`
3. Make the dock start a long reply:
   `curl -X POST localhost:8099/api/brain/anne-bot/think -H 'content-type: application/json' -d '{"text":"Tell me a long story about the moon, several sentences.","ttlMs":30000}'`
4. **While it's speaking, talk over it** (tap first if barge-in is tap-gated):
   say something clearly, e.g. "stop, what time is it?"
5. **Expect:** TTS stops mid-sentence; your new utterance becomes the next turn;
   the dock answers *your* question. Check the brain turn stream / obs.
6. **Negative test (no false self-barge):** make the dock speak a long passage and
   **stay silent** — it must NOT interrupt itself (AEC + no stray transcript).
   Watch `/api/perception/snapshots?dock=anne-bot` — zero speech during its own TTS.
