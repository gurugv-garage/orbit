## What's under test (this WIP)

**The purpose-rule** (docs/decision-traces/behaviour-conductor.md): producing /
maintaining memory+perception **artifacts** is the perception PIPELINE's job; a
behaviour/task's main goal is to **act** (it may touch memory as a side effect). That
rule moved memory *curation* out of a "daydream task" and into the pipeline as the
**memory curator**. What survives here:

| Piece | Code | Kind |
|---|---|---|
| **Task memory (DIRECT)** `this.memory` | `tasks/_harness/memory.ts` (`DockMemory`) | dock-scoped direct `MemoryStore` access — NOT a wire capability (memory = shared code + env, not live state) |
| **Long-term memory curator** (PIPELINE, not a task) | `modules/perception/memory/longterm/` | consolidate (create) + reconcile (revise/forget); load-aware cadence; restart-safe watermark |
| Curator console surface | `perception/index.ts` `/curator*`, `PerceptionStudio.tsx` | toggle + recent-passes feed + run-now |
| `MemoryStore.docks()` + `geminiText()` helper | `memory/store.ts`, `summarizer.ts` | support for the curator |

**Rehomed:** the `daydream` task was deleted; its curation logic is now the curator
above (purpose-rule: maintaining memory ⇒ pipeline). `create-artifact` was dropped.
The memory **wire capability** was also removed — memory is reconstructible from shared
code + `.env`, so tasks reach it **directly** (`this.memory`); the wire is reserved for
the station's live in-process state (frame/move). See [tasks.md](../tasks.md) "Direct
vs. the wire".

NOT yet under test (later phases): the conductor itself, the actuator lease,
faceFollow, barge-in, ResourceMonitor, deterministic raw→summary→layer building.

## 👤 MANUAL (live station + real LLM)

**Preconditions:** station up (`npm run dev`), `.env` has `GEMINI_API_KEY`. The
long-term curator runs **in-process** and **auto-fires** (no `run_task`), master-gated by
`PERCEPTION_CURATE` (on unless `=0`). Easiest manual driver: the **🧹 curator panel** in
the Perception Studio (toggle, **run-now**, per-pass feed) or `POST /api/perception/curator/run`.

- **reconcile** (maintain): seed beliefs via the console inspector / `POST /api/perception/memory`
  (`{dock,type,claim,subject?,confidence?}`); run-now; inspect via the memory REST routes.
- **consolidate** (create): needs **real speech** at a dock (diarized source). The
  cadence knobs (`PERCEPTION_CONSOLIDATE_AT|MAX_AGE_MS|QUIET_MS|MAX_BATCH`) can be lowered
  for a fast manual run.
