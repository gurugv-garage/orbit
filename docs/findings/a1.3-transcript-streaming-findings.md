## State today (done)

- The station's `transcript` perception frame already renders in the phone's
  subtitle band (`DockScreen.onPerceptionFrame` "transcript" case →
  `PerceptionEvent.Transcript(final=true)` → `PerceptionWiring`). So the user's
  words DO appear on the dock — one clean line per utterance, right after they
  stop speaking (the VAD endpoint). This is the A1.2 wiring; it works.

## Why partials are NOT cheap (the architecture)

The STT is **utterance-final by design**: `stt-watch` VAD-endpoints an utterance,
then sends the WHOLE thing to the whisper sidecar **once** (`/transcribe {pcm} →
{text}`, one-shot, no streaming). This is *deliberate* — transcribing rolling
windows made Whisper hallucinate/loop (the documented "Whisper wasn't bad, it was
mis-fed" finding). So there is no interim/partial signal to forward.

True word-streaming would need one of:
- **Interim decode passes** — re-transcribe the growing in-progress buffer every
  ~300–500 ms. Costs N extra whisper calls per utterance + re-introduces exactly
  the rolling-window hallucination the VAD design fixed.
- **A streaming-capable STT** — swap MLX-whisper one-shot for a streaming model.
  A real component change.

Both are real work, and the plan itself says do this "only if the latency/cost is
acceptable; otherwise keep final-only." It is **not** acceptable cheaply.
