# Recall & capture reliability — making the conversation less dumb

> **Goal:** when you ask the dock about your recent time together, it answers
> accurately — and when it doesn't actually know, it says so instead of confabulating.
>
> **The key reframe:** recall reliability is *downstream of capture* reliability. If
> the STT mishears and the video snapshots misdescribe the scene, the stored "facts"
> are already wrong — no recall tuning fixes garbage-in. Today the two are tangled:
> you can't tell whether a bad answer is **bad capture** or **bad recall**. So we
> **split them**, and fix **capture first**.
>
> Status: planning. The first build is the capture record/replay harness.

## The two halves (split them — this is the whole point)

### CAPTURE — is what we perceive accurate?  ← FOCUS FIRST
The continuous **background perception** — "what's happening when nobody's talking to
the dock." Two paths, both currently unreliable:
- **STT** — transcribes speech; imperfect.
- **Video snapshotting** — the every-few-seconds vision description of the scene;
  imperfect. (It's the *snapshotting* that's unreliable, not the later fusion summary.)

If capture is wrong, everything downstream inherits the error. Fix this first.

### RECALL — given accurate facts, can it retrieve + answer?
Tested in **isolation** so capture noise doesn't pollute it: inject **known** facts
straight into memory, then ask recall questions. A clean read on retrieval + synthesis
+ honesty, independent of whether perception captured correctly.

## How we measure CAPTURE — real A/V, recorded + replayed

**Benchmark = real recorded perception, played back.** Not synthetic, not text-injected.

| Decision | Choice |
|---|---|
| **First build** | A **record + replay harness**: capture real ambient A/V, replay it through the capture pipeline, score STT + video-snapshot accuracy, swap models on the *same* recording. |
| **Recording source** | Tap the dock's **live WebRTC stream** (the real production path — same mic/cam/conditions the dock actually sees). The station SFU already has a tap seam (`modules/media/tap.ts`, `ForwardingTap` records) — mostly a wiring job. |
| **Judge** | **Human-in-the-loop with LLM assist** — an LLM pre-scores capture-vs-reality to speed it up; the human is the final say (ambient-scene accuracy is partly subjective). |

**Architecture: record once, reprocess N times.** The recorder saves **raw A/V only**
(video.webm + audio.wav, with a start epoch). Processing is a **separate step** run
from the console: "process this recording with model X" → a named **result run**
(snapshots produced by re-running the REAL processors over the file). You can run many
(whisper-as-is, whisper+context-prompt, a different STT, vision model A/B, a Claude
summarizer) and **compare them side by side on identical input**. This is what makes
it a real model benchmark — not "what happened to be running when I recorded."

- **Reprocess = re-run the real processors on the file** (stt-watch / vision-snapshot
  with a swappable sidecar model) — same VAD/endpointing/snapshotting as production, fed
  from the recording instead of live RTP. Faithful, not a shortcut.
- All control on the **station console**: select dock → record-mode → start/stop →
  pick model(s) → process → compare → judge.

What the harness lets us measure (the real STT pain points):
- **Multi-speaker confusion** — without diarization, STT garbles overlapping voices.
  Record a multi-person clip, reprocess with different models/settings, see which is
  least wrong. (We're fine transcribing everyone — just need it *accurate*.)
- **Context-aware transcription** — better guesses when the model knows the context
  (names, domain terms). Test: reprocess with vs. without a context/prompt bias and
  compare. (Whether Whisper supports an `initial_prompt`/bias is a thing to confirm.)
- Build a small library of real recordings as a **regression set** for capture.

### The judging console (the human-in-the-loop tool)

A console in orbit-station to make capture judging *fast and visual*: play the
recorded **A/V with the snapshot timeline synced to playback time**. As the video
plays (or you scrub), it shows exactly **which vision snapshot + STT transcript were
produced at that moment**, aligned in time — so you can judge "did capture match
reality *right here*."

- **Transport:** play recorded audio + video; a scrubber/timeline.
- **Synced tracks:** under the playhead, the vision-snapshot description and the STT
  transcript for the current time (the timeline shows each snapshot as a marker at its
  timestamp; the active one highlights as you play).
- **Judge controls:** mark each snapshot/transcript ✓ accurate / ✗ wrong (+ optional
  note); an LLM pre-scores to pre-fill, human confirms/overrides.
- **Model compare:** replay the same recording under model B, see its snapshots on the
  same timeline next to model A's for the same instant.
- Lives alongside the existing Perception Studio (`/#perception`) — same snapshot
  format, just time-synced to recorded A/V instead of live.

## How we measure RECALL — known facts, in isolation

Inject a set of **known facts** into the memory store at known times, then ask a fixed
list of recall questions. Score **correct / wrong / confabulated** (honesty is its own
axis — a confident wrong answer is worse than "I don't know"). Because the facts are
known-good, any failure is *recall's* fault, not capture's.

## Build order

1. **Recorder** (server, building): station-controlled, dock-selected start/stop;
   records raw A/V (webm+wav) off the live WebRTC stream; **recording mode** so the
   dock doesn't respond while capturing. Artifact = raw A/V + start epoch.
2. **Reprocess** (server): "process recording R with model X" → re-run the real
   stt-watch / vision-snapshot processors over the file → a named **result run**.
3. **Console** (web `/#capture`): dock selector + record controls; pick model(s) +
   process; **A/V playback with the result's snapshot timeline synced to the
   playhead**; **compare** runs side by side; **judge** marks (human + LLM-assist).
4. **Use it** to measure + improve STT (multi-speaker, context-aware) + vision.
5. **Recall isolation harness** (separate, simpler): inject known facts → score recall.
6. **Fixes**, prioritized by what the harnesses show.
